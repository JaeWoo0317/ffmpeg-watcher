## ✅ 필요한 것 (먼저 설치해야 해요!)

### 1단계. Node.js 설치

> Node.js는 이 프로그램을 실행하기 위해 꼭 필요해요.

1. 👉 [https://nodejs.org](https://nodejs.org) 에 접속합니다.
2. 초록색 **LTS** 버튼을 클릭해서 설치 파일을 다운로드합니다.
3. 다운로드된 파일을 더블클릭해서 설치합니다.
   중간에 나오는 선택지는 모두 **Next(다음)** 을 눌러도 됩니다.
4. 설치가 끝나면 **컴퓨터를 재시작**합니다.
5. 재시작 후 설치가 잘 됐는지 확인합니다:
   - 키보드에서 `윈도우 키 + R` 누르기 → `cmd` 입력 → 확인
   - 검은 창이 열리면 아래 명령어를 입력하고 Enter:
     ```
     node -v
     ```
   - 아래처럼 버전 숫자가 나오면 성공이에요! ✅
     ```
     v22.0.0
     ```

### 2단계. FFmpeg 설치

> FFmpeg는 실제로 영상을 변환해주는 도구예요.

#### 방법 1 — winget 사용 (가장 쉬움, 권장)

1. 검색창에 `cmd` 입력합니다.
2. 검색 결과에서 **명령 프롬프트**를 **오른쪽 클릭** → **관리자 권한으로 실행** 클릭합니다.
3. 아래 명령어를 입력하고 Enter:
   ```
   winget install ffmpeg
   ```
4. `Y` 또는 `예`를 누르고 설치가 끝날 때까지 기다립니다.
5. 새 명령 프롬프트 창을 열고 아래 명령어로 확인합니다:
   ```
   ffmpeg -version
   ```
   버전 정보가 쭉 나오면 성공이에요! ✅

#### 방법 2 — 직접 설치

1. 👉 [https://www.gyan.dev/ffmpeg/builds/](https://www.gyan.dev/ffmpeg/builds/) 에 접속합니다.
2. **ffmpeg-release-essentials.zip** 을 찾아서 클릭해 다운로드합니다.
3. 다운로드된 zip 파일을 `C:\ffmpeg` 폴더에 압축 해제합니다.
   - 압축 해제 후 `C:\ffmpeg\bin\ffmpeg.exe` 파일이 있어야 해요.
4. **시스템 환경 변수 PATH**에 추가합니다:
   - 검색창에 `환경 변수` 입력 → **시스템 환경 변수 편집** 클릭
   - 오른쪽 아래 **환경 변수** 버튼 클릭
   - 아래쪽 **시스템 변수** 목록에서 `Path` 선택 → **편집** 클릭
   - **새로 만들기** 클릭 → `C:\ffmpeg\bin` 입력
   - **확인** → **확인** → **확인** 클릭
5. 명령 프롬프트를 **새로 열고** 아래 명령어로 확인합니다:
   ```
   ffmpeg -version
   ```
   버전 정보가 나오면 성공이에요! ✅

## 📥 프로그램 다운로드

### 방법 1 — ZIP으로 다운로드 (가장 쉬움)

1. 👉 [https://github.com/JaeWoo0317/ffmpeg-watcher](https://github.com/JaeWoo0317/ffmpeg-watcher) 접속
2. 초록색 **`< > Code`** 버튼 클릭
3. **Download ZIP** 클릭
4. 다운로드된 ZIP 파일을 원하는 곳에 압축 해제합니다.
   예: `C:\Users\내이름\Desktop\ffmpeg-watcher`

### 방법 2 — Git으로 다운로드

명령 프롬프트에서:
```
git clone https://github.com/JaeWoo0317/ffmpeg-watcher.git
```

   > 👉 **http://localhost:4001**

| 항목 | 설명 | 예시 |

### 🎬 영상 탭

| 항목 | 설명 | 기본값 |
|------|------|--------|
| **코덱** | 영상 압축 방식 | H.264 |
| **컨테이너** | 파일 형식 (확장자) | MP4 |
| **프로파일** | H.264/H.265 품질 프로파일 | High |
| **해상도** | 출력 영상 크기 | 원본 유지 |
| **FPS** | 초당 프레임 수 | 원본 유지 |
| **비트레이트 모드** | 품질 조절 방식 | CRF |
| **CRF 값** | 낮을수록 고화질, 높을수록 작은 파일 (0~51) | 23 |
| **인코딩 속도** | 느릴수록 파일이 작아짐 | medium |
| **GPU 가속** | 그래픽카드로 빠르게 변환 | 자동 감지 |
| **병렬 인코딩** | CPU 코어를 나눠서 동시에 변환 | 꺼짐 |
| **파일 크기 제한** | 출력 파일 최대 크기 (MB) | 제한 없음 |
| **디인터레이스** | 인터레이스 영상을 부드럽게 변환 | 꺼짐 |
| **노이즈 제거** | 영상의 잡티 제거 | 꺼짐 |

**코덱 선택 가이드:**
| 코덱 | 언제 사용? |
|------|----------|
| **H.264** | 가장 호환성 높음, 어디서든 재생 가능 👍 |
| **H.265** | H.264보다 파일 크기 절반, 최신 기기에서 재생 |
| **AV1** | 가장 작은 파일, 변환 속도 느림 |
| **Copy** | 재인코딩 없이 컨테이너만 변경, 가장 빠름 |

## 🚀 사용하기

1. **감시 폴더** 경로를 설정하고 저장합니다.
2. 탐색기에서 감시 폴더를 열고, 변환하고 싶은 영상 파일을 **안에 넣습니다**.
3. 브라우저 하단 **실시간 로그** 창에서 변환 진행 상황을 확인합니다!

```
[10:30:01] 감시 시작: C:/Videos/input
[10:30:45] 감지됨: my_video.mp4
[10:31:20] 완료: my_video.mp4 → my_video.mp4  ✅
```

4. 변환 완료 후:
   - 📂 **출력 폴더** → 변환된 파일
   - 📂 **감시 폴더/done/** → 원본 파일 (자동 이동)

## ❓ 자주 묻는 질문

**Q. 브라우저가 자동으로 안 열려요!**
A. 브라우저 주소창에 직접 입력하세요: `http://localhost:4001`

**Q. "node는 내부 또는 외부 명령..." 오류가 나요!**
A. Node.js 설치 후 컴퓨터를 **재시작**하지 않은 것입니다. 재시작 후 다시 시도하세요.

**Q. "ffmpeg를 찾을 수 없습니다" 오류가 나요!**
A. FFmpeg가 설치되지 않았거나 PATH 설정이 안 된 것입니다. 위의 FFmpeg 설치 방법을 다시 따라해주세요. 설치 후 **새 명령 프롬프트**를 열고 `ffmpeg -version`을 다시 확인하세요.

**Q. 변환이 시작되지 않아요!**
A. 아래를 확인하세요:
- 감시 폴더 경로가 올바른지 확인 (`\` 대신 `/` 사용 권장)
- `done` 폴더 안에 파일을 넣지는 않았는지 확인
- 지원하는 영상 파일인지 확인 (`.mp4`, `.mkv`, `.avi`, `.mov`, `.wmv`, `.flv`, `.webm`, `.m4v`, `.ts`)
